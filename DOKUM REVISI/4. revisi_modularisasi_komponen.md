# Penjelasan Revisi: Modularisasi Komponen UI (Pemisahan Page)

Dokumen ini berisi rincian spesifik mengenai kode asal (sebelum direvisi) di mana struktur navigasi masih bercampur dengan struktur dasar halaman, dan menjadi seperti apa kodenya setelah komponen navigasi dipisah ke file khusus (Modularisasi).

Semua "Kode Asal" di bawah ini sebelumnya menumpuk menjadi satu file panjang di dalam `src/app/(karyawan)/layout.tsx`.

---

## 1. Pembersihan File `layout.tsx`
File ini adalah kerangka dasar (HTML/Body) untuk seluruh halaman dasbor karyawan. Sebelumnya, *layout* ini bertugas terlalu berat karena juga mengurus *rendering* navigasi atas (Navbar), tombol *Logout*, dan laci menu untuk HP (*Mobile Drawer*).

**🔴 Kode Asal (di `layout.tsx`)**
Kodenya sangat padat dan panjang karena seluruh struktur `<nav>` ditulis langsung di sini:
```tsx
  return (
    <html lang="id">
      <body>
        {user && (
          <nav className={styles.nav}>
            <Link href="/dashboard" className={styles.logo}>
              Reimbu<span>S</span>
            </Link>
            {/* Desktop Nav */}
            <div className={styles.navRight}>
              <span className={styles.userGreeting}>
                Halo, <strong>{userName}</strong>
              </span>
              <LogoutButton />
            </div>
            {/* Mobile Hamburger */}
            <MobileNav userName={userName} logoutButton={<LogoutButton />} />
          </nav>
        )}
        {children}
      </body>
    </html>
  )
```

**🟢 Kode Baru (di `layout.tsx`)**
Sekarang kodenya sangat bersih, elegan, dan mudah dibaca (menganut prinsip *Best Practice*). Kita cukup meng-*import* komponen `<Navbar>` saja:
```tsx
import Navbar from '@/components/Navbar'

  return (
    <html lang="id">
      <body style={{ margin: 0, padding: 0, backgroundColor: '#0b0f19', color: '#f8fafc', fontFamily: "'Inter', system-ui, sans-serif", overflowX: 'hidden' }}>
        {user && <Navbar userName={userName} />}
        {children}
      </body>
    </html>
  )
```

*(Catatan: Inline style pada `<body>` tetap dipertahankan karena beberapa properti seperti `margin`, `padding`, dan `overflowX: hidden` belum tercakup sepenuhnya di `globals.css` pada tahap ini, sehingga ditulis inline untuk mencegah layout shift).*

---

## 2. Pembuatan Rumah Baru `src/components/Navbar.tsx`
Kodingan navigasi yang dipotong dari `layout.tsx` tadi tidak dibuang, melainkan dibuatkan "rumah baru" di dalam folder khusus komponen.

**🟢 Kode Baru (di `Navbar.tsx`)**
Komponen ini di-*export* agar bisa dipanggil berulang kali di mana pun kita butuhkan, tanpa perlu menulis ulang struktur desainnya:
```tsx
import React from 'react'
import Link from 'next/link'
import LogoutButton from './LogoutButton'
import MobileNav from './MobileNav'

interface NavbarProps {
  userName: string
}

export default function Navbar({ userName }: NavbarProps) {
  return (
    <nav className="sticky top-0 z-50 flex h-[76px] items-center justify-between border-b border-slate-800 bg-[#0b0f19]/90 px-5 md:px-10 backdrop-blur-md box-border">
      <Link href="/dashboard" className="text-2xl font-bold leading-none text-slate-50 no-underline">
        Reimbu<span className="text-blue-500">S</span>
      </Link>
      
      {/* Desktop Nav */}
      <div className="hidden md:flex items-center gap-5">
        <span className="text-[15px] text-slate-400">
          Halo, <strong className="font-semibold text-slate-50">{userName}</strong>
        </span>
        <LogoutButton />
      </div>
      
      {/* Mobile Hamburger */}
      <MobileNav userName={userName} logoutButton={<LogoutButton />} />
    </nav>
  )
}
```

## 3. Pemindahan Komponen Pendukung
Selain membuat `Navbar.tsx`, komponen-komponen UI kecil pendukung lainnya yang dulunya berserakan di dalam direktori `src/app/` juga telah **dipindahkan seluruhnya ke dalam `src/components/`** agar lebih terpusat:
- `LogoutButton.tsx` (Tadinya berada di folder `dashboard`)
- `MobileNav.tsx` (Tadinya berada di folder `(karyawan)`)

**Tujuan Keseluruhan:**
Memastikan pemisahan tanggung jawab (*Separation of Concerns*). Folder `app` murni hanya mengurus jalur (*routing*) halaman, sedangkan folder `components` murni mengurus bongkah kepingan antarmuka (UI).
