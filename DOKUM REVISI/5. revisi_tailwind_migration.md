# Penjelasan Revisi: Migrasi ke Tailwind CSS v4

Dokumen ini menjelaskan proses *refactoring* tampilan antarmuka (UI) Karyawan dari yang awalnya menggunakan CSS Modules manual (CSS biasa) menjadi menggunakan framework utilitas modern **Tailwind CSS v4**.

---

## 1. Tujuan Migrasi
1. **Membersihkan Struktur File:** Mengurangi jumlah file di dalam proyek agar tidak berantakan dengan file `.module.css`.
2. **Mempercepat Pengembangan:** Menghapus kebutuhan untuk berpindah bolak-balik antara file TypeScript (TSX) dan file CSS saat melakukan *styling*.
3. **Meningkatkan Responsivitas:** Mengganti logika *media queries* manual (`@media max-width...`) dengan *prefix* responsif bawaan Tailwind seperti `md:` dan `lg:`.

---

## 2. Proses Instalasi (Tailwind v4)
Kita langsung menggunakan versi paling mutakhir dari Tailwind (v4) yang menawarkan konfigurasi minimal (Zero Configuration). 

Perubahan kunci:
- File `tailwind.config.ts` tidak lagi diperlukan dan **telah dihapus**.
- Integrasi *engine* PostCSS diperbarui dengan meng-install `@tailwindcss/postcss`.
- File `src/app/globals.css` dirombak murni hanya menggunakan import utilitas standar Tailwind:
  ```css
  @import "tailwindcss";

  body {
    background-color: #0b0f19;
    color: #f8fafc;
    font-family: ui-sans-serif, system-ui, sans-serif;
  }
  ```

---

## 3. Penghapusan File Usang (Beban Berkurang)
Dengan beralih ke Tailwind, kita telah **menghapus secara permanen** 4 file CSS manual dengan total ratusan baris kode:
1. ❌ `src/app/(karyawan)/layout.module.css`
2. ❌ `src/app/(karyawan)/mobileNav.module.css`
3. ❌ `src/app/(karyawan)/dashboard/dashboard.module.css`
4. ❌ `src/app/(karyawan)/dashboard/new/new.module.css`

---

## 4. Contoh Perubahan Kodingan (Before & After)

Berikut adalah contoh bagaimana kode menjadi lebih bersih dan deskriptif.

### A. Tombol Logout (`LogoutButton.tsx`)
**Sebelum (CSS Modules):**
```tsx
// TSX
<button className={styles.logoutBtn}>Keluar</button>

// CSS Terpisah
.logoutBtn {
  padding: 10px 20px;
  background-color: rgba(239, 68, 68, 0.1);
  color: #ef4444;
  transition: all 0.2s;
}
.logoutBtn:hover {
  transform: translateY(-1px);
}
```

**Sesudah (Tailwind):**
```tsx
// Semua properti styling ditulis langsung sebagai kelas utilitas
<button className="px-5 py-2.5 bg-red-500/10 text-red-500 transition-all duration-200 hover:-translate-y-[1px]">
  Keluar
</button>
```

### B. Form Input Kustom (`new/page.tsx` & `edit/[id]/EditForm.tsx`)
Untuk mencegah pengulangan *class* Tailwind yang terlalu panjang pada elemen input form (seperti *border*, efek *focus*, dan *padding*), kita menyimpannya ke dalam satu variabel JavaScript konstan:
```tsx
const inputClass = "px-4 py-3 bg-slate-950 border border-slate-700 rounded-lg text-slate-50 font-sans transition-all duration-200 focus:outline-none focus:border-blue-500 focus:ring-4 focus:ring-blue-500/10 w-full"

// Penggunaan
<input type="text" className={inputClass} />
<textarea className={inputClass} />
```
