# Penjelasan Revisi: Pemisahan Handler Form

Dokumen ini berisi rincian spesifik mengenai kode asal (sebelum direvisi) dan menjadi seperti apa kodenya setelah dipindah ke file service (handler) baru.

Semua "Kode Asal" di bawah ini sebelumnya menumpuk di dalam fungsi `handleSubmit` pada file antarmuka (UI) yaitu `EditForm.tsx` dan `new/page.tsx`.

---

## 1. `claimHandlers.ts`
File baru ini dibuat khusus untuk menampung seluruh logika pemanggilan API (seperti *upload* foto dan penyimpanan ke *database*). Analogi file ini adalah sebagai "Koki" di dapur.

**🔴 Kode Asal (di `EditForm.tsx` dan `new/page.tsx`)**
Kodenya ditulis menyatu dengan komponen React sehingga membuat file sangat panjang:
```typescript
  // ... potongan kode asal yang menyatu dengan UI
  if (file) {
    const formData = new FormData()
    formData.append('file', file)
    formData.append('_payload', JSON.stringify({ alt: `Nota Klaim Update - ${klaim.claimCode}` }))

    const mediaRes = await fetch('/api/media', { method: 'POST', body: formData })
    const mediaData = await mediaRes.json()
    if (!mediaRes.ok) throw new Error('Gagal mengunggah foto nota baru.')
    mediaId = mediaData.doc.id
  }

  const claimRes = await fetch(`/api/reimbursements/${klaim.id}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ category, itemName, description, amount, receipt: mediaId }),
  })
```

**🟢 Kode Baru (di `claimHandlers.ts`)**
Logika kasarnya diekstrak menjadi fungsi rapi yang bisa di-eksport:
```typescript
/**
 * Memperbarui data klaim reimbursement yang sudah ada.
 */
export async function submitEditClaim(params: EditClaimParams) {
  const { klaimId, claimCode, existingReceipt, file, category, itemName, description, amount } = params

  let mediaId = typeof existingReceipt === 'object' ? existingReceipt?.id : existingReceipt

  // Upload gambar baru jika ada
  if (file) {
    mediaId = await uploadMedia(file, `Claim Receipt Update - ${claimCode}`)
  }

  // Perbarui data klaim
  const claimRes = await fetch(`/api/reimbursements/${klaimId}`, {
    method: 'PATCH',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ category, itemName, description, amount, receipt: mediaId }),
  })
  // ... (error handling return logic)
}
```

*(Catatan: Di file ini juga terdapat fungsi `submitNewClaim` untuk form pengajuan baru, dengan format yang serupa).*
