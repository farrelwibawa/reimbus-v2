# Penjelasan Revisi: Penambahan Custom Component (Status Label)

Dokumen ini berisi rincian spesifik mengenai kode asal (sebelum direvisi) dan menjadi seperti apa kodenya setelah ditambahkan komponen kustom (Custom Element).

Semua "Kode Asal" di bawah ini sebelumnya hanya menampilkan teks mentah di Admin Panel. Kita menyulapnya menjadi *badge* warna-warni yang rapi dan interaktif.

---

## 1. `StatusCell.tsx`
File baru ini dibuat khusus untuk mendesain tampilan UI berupa label warna-warni (hijau, merah, biru, kuning) berdasarkan kata status yang tersimpan di *database*.

**🔴 Kode Asal (Tidak Ada)**
Sebelumnya kita tidak memiliki kodingan ini, sehingga Payload CMS merender teks status sebagai tulisan biasa (hitam putih).

**🟢 Kode Baru (di `src/components/StatusCell.tsx`)**
```typescript
'use client'
import React from 'react'

export const StatusCell = ({ cellData }: { cellData: string }) => {
  if (!cellData) return null

  // Fungsi penentu warna berdasarkan status
  const getColors = (status: string) => {
    switch (status) {
      case 'approved': return { bg: '#dcfce7', color: '#166534' } // Hijau
      case 'rejected': return { bg: '#fee2e2', color: '#991b1b' } // Merah
      case 'paid':     return { bg: '#dbeafe', color: '#1e40af' } // Biru
      case 'pending':
      default:         return { bg: '#fef9c3', color: '#854d0e' } // Kuning
    }
  }

  const { bg, color } = getColors(cellData)

  return (
    <span
      style={{
        backgroundColor: bg,
        color: color,
        padding: '4px 12px',
        borderRadius: '99px', // Membuat sudut melingkar oval (badge)
        fontWeight: 'bold',
        fontSize: '12px',
        textTransform: 'uppercase',
        display: 'inline-block',
      }}
    >
      {cellData}
    </span>
  )
}
```

## Hasil Akhir di File Konfigurasi Backend

Supaya komponen di atas bisa muncul secara ajaib di tabel Admin Panel, kita harus "mendaftarkannya" di file konfigurasi Payload.

**🔴 Kode Asal (Before)**

*Di file `adminFields.ts`:*
```typescript
export const statusField: Field = {
  name: 'status',
  type: 'select',
  defaultValue: 'pending',
  options: statusOptions,
}
```

*Di file `Reimbursements.ts`:*
```typescript
export const Reimbursements: CollectionConfig = {
  // ...
  admin: {
    useAsTitle: 'claimCode',
  },
}
```

**🟢 Kode Baru (After)**

*Di file `adminFields.ts` (Mendaftarkan UI kustom agar dipakai oleh field status):*
```typescript
export const statusField: Field = {
  // ...
  admin: {
    components: {
      Cell: '@/components/StatusCell#StatusCell', // <-- Memanggil komponen UI baru kita
    },
  },
}
```

*Di file `Reimbursements.ts` (Menampilkan kolom status secara default di tabel terkanan):*
```typescript
export const Reimbursements: CollectionConfig = {
  // ...
  admin: {
    useAsTitle: 'claimCode',
    defaultColumns: ['claimCode', 'category', 'itemName', 'description', 'status'], // <-- Status dimasukkan ke list
  },
}
```
